《圣都布尔乔》

我是狗
月亮升起来的时候，我就要开始嚎叫，我也不知道我为什么要叫，但是体内总有一种野性在升腾,那可能是远古祖先留给我的血脉，像根植在我血液中的一条通天巨藤，每次月圆夜，我就要沿着这条藤往上爬，爬到和天齐高的地方。没什么可受质疑的，血脉是最牢不可破的誓言，也可以把它当成一种暗示，一种召唤。
我已经感受到这种召唤了。

我是吉秋
你见过一株金盏花被照亮的样子吗，一开始的世界是暗淡的，眼睛由于长时间盯着昏暗的地方而愈发疲惫，就当你觉得它快要丧失光感的时候，世界就那么一小点、一小点地亮起来了。先是黝黑的土地被镀上一层银色光晕，再是托举花朵的茎叶，然后就是花瓣了，银色的光辉从这一瓣流淌到下一瓣，花瓣间促拢像酒杯似的凹陷盛满月光，带来了金色，红得黯淡，金得耀眼，整个世界就在你面前展开了。
在等待柴火将晚饭烧热的间隙，我经常这样观察这朵花，这对我来说像个好玩的游戏——趁着太阳落山那一刻把柴火备好，切开，在锅下面叠整齐，点火，然后等待月光照亮长在栅栏这旁的金盏花，挪步回去，慢悠悠地加入一早切好的胡萝卜和土豆块，我喜欢观察土豆在锅里越煮越黄的样子、胡萝卜将自己的颜色融入汤里的过程，然后端进屋里，叫自己的师傅皮尔切和他的妻子德塔以及三个不知道刚从哪个泥巴坑里翻滚出来的孩子上桌吃饭。
他们一定不会发现，自己吃饭的时间竟是由门前栅栏旁那朵毫不起眼的金盏花决定的。这种掌控感让我感觉窃喜。
皮尔切是村子里最好的木工，有一天他在给诺尔家做桌子的时候被锯子割伤了手，随即流出来的鲜红的血让他有了一种被背叛的感觉：被这把他常年累月使用的最熟悉的锯子背叛，被时间背叛。这种感觉让他找到了我父亲，说想要收我做他的学徒。父亲那时候正在为今年欠收的谷子地发愁，他一手撑在锄头上，一手抹了抹自己鼻尖的汗，有些窘迫地说：“那个……我很乐意让我的儿子去学点什么，有个手艺在这个世道真是一件最幸福不过的事。只是我们家已经快吃不起饭了，实在是没有什么钱，没办法缴纳那笔学徒费用。”
“要不您看这样……”
“没什么，我还可以给你十捆柴火。”皮尔切很快地打断了他，“我也是看你们是村里最穷的人家，想帮帮你们。这样，先让他在我这干活几年，一边干活，我一边教他手艺，学徒费一分不收。但条件是，等他出师，得先为我免费做工三年。”
父亲在脑海里盘算起来，但是一个下午的耕作已经让他的脑袋里由汗液粘成了一团浆糊，做工，免费，这些词他一概也没听清，只是想着自己的儿子能学到一手木工手艺，以后可以代替皮尔切在村子里接活，等多干几年熟练以后，说不定会把生意做到村外，做到市里也不一定。那他们世世代代种谷子的命运算是在他这断绝了，没有比这更好的交易，他装模作样地思索了一阵后同意了，浑然不觉其实这样只是免费卖掉了一个快要身强力壮的劳力，甚至比昨天卖掉的那只五十比索的老年母牛还要便宜。在几年后，他开始感到对农活力不从心而腰酸背痛的无数个日夜里，他经常会这么想“我的儿子甚至不如一只五十比索的牛，就这样被我卖给了皮尔切。该死的皮尔切，该死的厄运，只会欺负我这样靠双手吃饭的可怜人。”这样的想法一直持续到我失踪不见，持续到他的不如意的老年。
在这个村子里，没有人会刻意养花，这是瑞玛王朝里最穷也最微不足道的一个村子，如果把瑞玛王朝广阔的土地看成是一个鸡蛋，那这个村子就是鸡蛋下半部壳上一个最普通不过的气孔，可能是一群旅人依着小溪建立起一个营寨，但要知道，在长途跋涉中最能阻挡人脚步的可能不是突然出现的强盗野兽，也不是有了上顿没下顿的饥荒，而是突如其来的放松和休息，几个旅人扎营建寨，就着篝火的光亮回忆往昔，劣质酒精像一把锤头，锤散了他们的斗志，从此认命，想着这个地方又靠近小溪，又是平原，离一个规模不大不小的镇子路途也不远，酒精和火光烘托出一种宿命感，不如就当定居下来的第一群人吧。
鸡蛋王朝的下半部亮起一丝光亮，又很快变成了一户，几户，十几户，几十户。可能会继续扩大，也可能会很快熄灭。在这天高皇帝远的地方，一个普通地主可能比住在鸡蛋北部，从不踏出皇宫之外的王更有势力。忠诚只是一种宗教，人们左右盘旋，看看哪个宗教供奉的神更能给予精神的安定。
这个村没有人会养花，村民把对生活的热情全部献祭给了活着的每一时刻：种地，学一门手艺，干活，煮饭，生孩子……一点不剩。
当然，除了我，我是吉秋，我见过一株金盏花被月亮照亮的样子。

我是逃离
很多人都会在生活的某个时刻想到我，可是很多人也都会误会我。
我乔装成很多种样子：一阵由近及远的马蹄声，一顿还没有下锅的晚饭，一段暴躁如雷的争吵，一句突如其来的抱怨……我喜欢潜伏在这些瞬间，可是我没什么耐心，很快我又会离开，因为我知道这里面的大部分人都缺乏勇气。

（突然的灵感：逃离认为有勇气的人它会给予他们一点什么，这本书可以误导为找到圣都布尔乔彻底改变命运或者改变这个时代，但其实逃离给予的是主角在路上的各种细节：和朋友的信任，争吵，微醺的夜晚，对明天的希冀，一路看到的美景……这就是逃离给予他的，全新的生活。毕竟到了布尔乔后，什么都不会发生，这是这个时代注定的结局，个人的努力挽不回整个时代的落后。）

常用http状态码

分类：

1. 信息类 100预请求成功，可继续。 101切换协议
  1. 成功 200成功，201创建了资源。202接受但没处理完。204处理成功但无返回。206成功处理部分。
  2. 重定向 301已永久移动到新地址，302临时移动到新地址。304请求的资源未修改，服务器不返回资源，可继续用缓存。305代理。
  3. 客户端。400 客户端语法错误，服务器无法理解。401 未登录，要身份验证。403 拒绝执行。404找不到资源。405 方法被禁止。
  4. 服务器端 500服务器内部错误，无法完成请求。常见异常。 501不支持功能。502网关错误。 503超时。504网关超时。505 http协议超时。

这类码，需要经常接触才能熟悉。

参考：https://www.runoob.com/http/http-status-codes.html

Regular Expression 正则表达式 (一)

正则表达式用于字符串查找和匹配。此篇用于基础，后续会更新进阶。

先自行思考接触过的正则表达式，然后再根据下文查漏补缺。

元字符

\d 匹配数字

\w 匹配字母或数字或下划线或汉字

\s 匹配空白字符

^ 开始符号

$ 结束符号

\b 匹配单词开始或结束

重复限定符

. 匹配任意一个字符(换行符除外)

\ 反斜杠转义

？ 匹配0次或一次

* 匹配0次或多次

+ 匹配1次或多次

{n} 重复n次

{n, } 重复n次或更多次

{n, m} 重复n-m次

分组

（） 小括号括起来为一组

或

｜ eg: ^(189 | 180 | 177)\d{8} $

区间

[0-9]、[a-z]、[].

进阶知识点

零宽断言

断言： 指定内容前后会有满足规则的内容。

零宽：没有宽度，指指匹配位置，不匹配断言本身。

即零宽返回值都不包含以下例子中的pattern。

正向先行

?= pattern， 匹配pattern前面的内容。

正向后行

?<=pattern, 匹配pattern后面的内容

负向先行

?!pattern， 匹配非pattern的前面内容。

负向后行

?<!pattern, 匹配非pattern的后面内容。

捕获和非捕获

捕获一般与分组联系在一起。

数字编号捕获组

(exp) 表达式从左往右，遇到左括号就分一个组。第0组为整个表达式，但不计入分组数量。

(0\d{2})-(\d{8})

命名编号捕获组

(?exp) , 可以用命名来区分组。

(?\0\d{2})-(?\d{8})

非捕获组

(?:0\d{2})-(\d{8})

反向引用

\捕获组编号

基础正则表达式没有记忆能力，所以要用捕获和引用才能记住之前搜过的。

eg: 先捕获一个组，根据数字编号就可以再次用它。

(\w)\1

结束语

正则内容暂时就写到这，以前也只学过简单内容，没想到还有进阶内容。熟能生巧，正则这玩意儿还是得靠多多练习。

java基本概念

0001 JRE JDK JVM

  1. JDK java development kit， 包含JRE和开发所用，包括编译器，可以创建和编译。
  2. JRE java runtime environment, 包含jvm、java类库、java命令等，有就可以运行java程序。（只运行不创建）
  3. JVM java虚拟机，运行字节码的。字节码在jvm上运行，不同操作系统上的jvm实现不一样，但同一字节码不同jvm运行结果一致。

0002 面向对象是什么？和面向过程的区别。

面向对象是一种解决问题的思想，编程的思想。从原先只注重流程化的工作中抽象出来一个个对象，界定好对象的成员和职责。对象提供操作方法，我们只需要关注对象和对象之间关系就可以写代码。而对于面向过程，我们只注意具体流程设计，不太考率复用性和组件性。

0003 java语言特点

  1. 面向对象（封装、继承、多态）
  2. 一次编译、到处运行。平台无关
  3. 先编译成class, 再用jvm解释执行

0004 java和C++区别

  1. c++有指针
  2. java有自动内存回收机制gc，c++要手动
  3. c++可以多继承，java通过借口来实现多继承。

0005 java基本类型

  1. 占字节大小
    ：1248 248
    byte short int long char float double boolean
    不加static修饰的成员变量存放在堆中，局部变量在栈
  2. 装箱与拆箱 Integer.valueOf() Integer.intValue()
    包装类会有缓存范围，范围内则不会创建新对象，范围外则会新建对象。范围外值一样也不会equals.

0006 new一个对象发生的事情

  1. Stu a = new Stu();
    在堆上创建了一个Stu类的对象实例，在栈里创一个引用a,a指向对象实例。

0007 构造方法

  1. 构造方法没有返回值默认无参，可以重载（同名不同参）不能重写。

0008 面向对象的三大特征

  1. 封装、继承、多态
  2. 封装 对外展示是类的方法，内部属性隐藏，不允许直接修改。
  3. 继承 子类继承父类，所有的都会继承，但父类私有的不能动。子类可以自行增加或者重写。
  4. 多态 继承/实现+重写+父类引用指向子类对象
    eg : A a = new B(); B extend A
    eg : 实现接口 B implement A;
    对多态的理解：一处代码，却可以赋予多种实现，程序运行时才确定到底执行睡。

0009 接口和抽象类有什么区别？

  1. 都是用来声明某一类规则的，所以是虚的，不能实例化、可以有抽象方法，也可以有默认实现。
  2. 接口只要是行为约束，抽象类更多是面向对象的所属关系。
  3. 一个类可以实现多个接口，但是只能继承一个抽象类。
  4. 接口中的变量是固定的,public static final, 必须初始化并且不可改变，抽象类随意。（此处疑问？为什么要引入默认方法？–临时解答：为方便在原有接口上增加功能。）

0010 深拷贝、浅拷贝、引用拷贝

  1. 该问题的研究对象： 引用（变量）T——> A对象（B对象为内部对象）
  2. 深拷贝: TAB三者独立
  3. 浅拷贝: T和A都拷贝，比深要浅一层；
  4. 引用拷贝: 只拷贝引用T，AB不动

0011 hashcode equals

  1. equals > hashcode, 重写equals必须重写hashcode.

0012 String

  1. 字符类 是一个类而不是基本数据类型，所以要用对象的眼光去看待。
  2. 字符串，是具体String对象的值。常量池存储。
  3. String 不可变，是因为final修饰不能被继承+内部字符数组不能被继承也不提供对外修改方法的共同作用。所以线程安全。
  4. StringBuilder 可变 线程不安全

0013 异常
